# 미디어 아트 뷰어 구현 설명서

## 구현 변경 사항

### 1. 제너러티브 아트 엔진 (`js/generative.js`)
작가별 테마에 맞춘 5가지 고유한 절차적 생성(Procedural Generation) 엔진을 구현했습니다:

| 엔진 이름 | 설명 | 적용 작가 |
|--------|-------------|---------|
| **Organic (유기적)** | 정해진 경로 없이 부드럽게 흐르는 입자. 자연과 생체 발광을 표현. | AURA-7, FLORA-9, PRISM-2 |
| **Geometric (기하학)** | 회전하는 다각형과 깔끔한 선. 구조미와 여백의 미 강조. | KURO-X |
| **Cyberpunk (사이버펑크)** | 디지털 비(Rain)와 글리치 효과. 고대비 네온 비주얼. | NEON-V |
| **Wave (파동)** | 소리의 주파수나 유체 역학을 표현하는 사인파 합성. | ECHO-0, AQUA-5, TERRA-1 |
| **Cosmic (우주)** | 깊이감이 있는 Z축 별빛 시뮬레이션. | VOID-3 |

### 2. 모듈형 뷰어 로직 (`js/viewer.js`)
- `MediaArtViewer` 코드를 리팩토링하여 렌더링 로직을 각 `ArtEngine` 하위 클래스로 위임했습니다.
- `artists.json` 데이터를 `js/artists.js`(`ARTISTS_DATA`)로 변환하여 로컬 서버 없이 파일 실행만으로도 동작하도록 개선했습니다.
- 동적 로딩 로직 추가: 설정 `engineType`에 따라 적절한 엔진 인스턴스화.

### 3. 데이터 설정 (`js/artists.js`)
- 9명의 작가 프로필 데이터에 `engineType`과 테마 색상 팔레트 설정을 포함하여 JS 파일로 관리.

## 사용 방법

### 1. 실행 방법
이 프로젝트는 이제 웹 서버 없이도 실행 가능합니다.
- 탐색기에서 `index.html` 파일을 더블 클릭하여 브라우저로 엽니다.
- 또는 `npx serve .` 등의 로컬 서버를 이용해도 됩니다.

### 2. 감상 방법
1. **메인 화면**: 9명의 작가 카드가 나타납니다.
2. **작품 보기**: 카드를 클릭하면 해당 작가의 고유한 미디어 아트 뷰어가 실행됩니다.
   - **AURA-7**: 녹색 계열의 부드럽게 떠다니는 입자
   - **KURO-X**: 흑백의 회전하는 기하학 도형
   - **NEON-V**: 핑크/청록색의 디지털 비
   - **ECHO-0**: 다채로운 파동 패턴
3. **컨트롤**:
   - 마우스를 움직이면 컨트롤 버튼이 나타납니다.
   - `F11` 또는 전체화면 버튼: 몰입형 감상
   - `ESC` 또는 뒤로가기 버튼: 작가 목록으로 이동
